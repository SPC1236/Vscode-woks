üß† What Are Type Annotations?

Type annotations specify the expected data types for variables, function parameters, and return values.
They're optional but highly recommended for improving code clarity, documentation, and error checking using tools like mypy.

üìö Why Use Them?

  • Improves code readability and structure
  • Helps catch type-related bugs early
  • Supports robust API design (e.g., with FastAPI)

üî¢ Basic Type Annotations

Python supports annotations for built-in types:

Type Description Example
int Integer numbers age: int = 25
float Decimal numbers price: float = 19.99
str Text strings name: str = "Alice"
bool Boolean values is_active: bool = True

üìò When to Use Which Type

  • int: Use for whole numbers like age = 25 or count = 100
  • float: Use for decimal values or precise quantities like price = 19.99
  • str: Use for text like name = "Alice" or message = "Hello"
  • bool: Use for true/false flags like is_active = True
  • list: Use for ordered, changeable sequences like colors = ["red", "blue"]
  • tuple: Use for fixed-size collections like coordinates = (10.5, 20.3)
  • dict: Use for key-value pairs like {"name": "Bob", "age": 30}
  • set: Use when you need unique values only like {"apple", "banana"}
  • Any: Use when you're unsure of the type, or it can be anything ‚Äî like external input.

Quick Tip: Use list when order matters, dict for structured data, set when duplicates must be avoided Any for generic or flexible values.

üîÅ Function Annotations

Annotate parameters and return types for more reliable functions:

def greet(name: str) -> str:
    return f"Hello, {name}"
from typing import List

def average(scores: List[float]) -> float:
    return sum(scores) / len(scores)

‚ú® Something New: TypedDict

TypedDict allows you to define structured dictionaries with fixed key-value types.
It‚Äôs useful when modeling JSON-like data.

from typing import TypedDict

class UserProfile(TypedDict):
    name: str
    age: int
    is_active: bool
    balance: float
user: UserProfile = {
    "name": "Sarah",
    "age": 28,
    "is_active": True,
    "balance": 245.75
}

This ensures consistency and validation when passing around complex data.
